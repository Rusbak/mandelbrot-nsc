# L09 - Testing and Docstrings
L09 focuses on making scientific code trustworthy through testing, documentation, and reproducibility. It motivates the need for verification using real‑world failure cases and then introduces pytest for writing structured, automated tests. The topic designs testable functions, uses parametrization, and measures coverage. The topic also covers NumPy‑style docstrings, clarifying function purpose, parameters, and return types, and adds type hints to support static analysis. Tools like MyPy help detect inconsistencies early.
